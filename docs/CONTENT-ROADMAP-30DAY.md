## How the 150 articles intertwine

The site uses a **hub-and-spoke topical map**. Hubs are the anchor articles (the seeds + a small set of cornerstone articles per cluster). Spokes are deep-dive articles that link back to their hub and to 1–2 sibling spokes. This is how Google and the AI engines build a confident topic-authority signal: many narrow articles all reinforcing one umbrella claim.

```
                      ╔══════════════════════════════════╗
                      ║  REAL ESTATE MARKETING (anchor)  ║
                      ║  Hub: "Marketing in 2026"         ║
                      ╚════╤═════════════╤══════════════╤═╝
                           │             │              │
                ┌──────────▼────┐  ┌─────▼─────┐  ┌─────▼─────────┐
                │ LEAD          │  │ PERSONAL  │  │ PERSONAL      │
                │ GENERATION    │  │ BRANDING: │  │ BRANDING:     │
                │ Hub: "7       │  │ KNOWN     │  │ WIN BEFORE    │
                │ Sources..."   │  │ BEFORE    │  │ YOU ARRIVE    │
                │               │  │ NEEDED    │  │               │
                │ Hub: "Why     │  │ Hub: "Go- │  │ Hub: "What    │
                │ Cold Calling  │  │ To Agent  │  │ Community     │
                │ Is Killing"   │  │ in Town"  │  │ Market Leader │
                │               │  │           │  │ Means"        │
                └───────────────┘  └───────────┘  └───────────────┘
                      45 articles      22 articles      23 articles
                      across clusters
```

**Cross-pillar linking rules** — every article links to:

1. **Its hub** (1 link, top of body)
2. **2 sibling spokes** in the same cluster (mid-body)
3. **1 cross-pillar link** to a hub in a different pillar (where it fits naturally)
4. **The pillar landing page** (`/real-estate-marketing`, `/real-estate-lead-generation`, `/personal-branding-authority`)

That's a minimum of 5 internal links per article. The first 5 seed articles already follow this pattern.
